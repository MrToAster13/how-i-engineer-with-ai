# One competency, shown through provenance-routed evidence

This repo demonstrates exactly one competency — AI / agentic coding. A Project earns a
place here **only if it was genuinely built *with* AI/agentic coding**; any security or
GRC competency it also proves lives on the résumé site, and it is re-lensed here through
*how* it was built, not what it does. Real work done by hand — the detection-engineering
homelab — is deliberately out of scope here, because it is not evidence of *this*
competency, however good it is. Evidence splits into three
buckets — **Skills** (artifacts I Built or Modified), **Workflows** (practices), and
**Driven** (tools I operate but did not make) — and an item's provenance decides its
bucket. An item that is both a Skill and a Workflow (e.g. `grill-me`) is cross-listed in
each bucket it fits.

## Considered options

- **Fold the AI repos into the résumé site** as three project cards, a tooling track, or a
  single umbrella card. Rejected: it dilutes the site's SOC/detection focus and gives the
  AI-engineering story no room to be told on its own terms.
- **A broad multi-competency portfolio in this repo.** Rejected: a repo that tries to prove
  everything proves nothing sharply; one competency, deeply evidenced, is the stronger claim.

## Consequences

- Security/GRC depth stays on the résumé site; here, case studies lead with method, not domain outcome.
- Provenance labels (Built / Modified / Driven) are load-bearing — mislabeling one breaks the
  repo's entire honesty premise, so they get the same scrutiny as a factual claim.
